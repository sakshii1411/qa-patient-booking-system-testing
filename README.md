# QA Patient Booking System Testing

## Project Overview
This project demonstrates end-to-end API and Manual Testing of a Booking Management System using Postman.

The objective was to validate booking workflows including Authentication, Create, Read, Update, and Delete operations while verifying response codes, authorization, request validation, and error handling.

---

## Objective
- Perform API testing on a live booking application
- Validate CRUD operations
- Test authentication and authorization
- Verify API responses and status codes
- Practice real-world QA workflow using Postman

---

## Test Scenarios Covered
- Authentication Testing
- Create Booking
- Get Booking by ID
- Update Booking
- Delete Booking
- Verification after Deletion
- Negative Testing

---

## API Endpoints Tested
- POST /auth
- POST /booking
- GET /booking/{id}
- PUT /booking/{id}
- DELETE /booking/{id}
